# AutoPent: AI-Agentic Penetration Testing Framework

**Author:** Abhinav Mehra, Himanshu Gaur <br>
**Domain:** Cybersecurity, Offensive Security, Automated VAPT

AutoPent is a next-generation, autonomous penetration testing framework designed to bridge the gap between static vulnerability scanners and manual offensive security operations. By utilizing Large Language Models (LLMs) as a core reasoning engine, AutoPent orchestrates the entire cyber kill chain from initial reconnaissance to active exploitation and remediation.

---

## 1. Problem Statement
The modern cybersecurity landscape faces a critical bottleneck in vulnerability assessment and penetration testing:
* **Limitations of Automated Scanners:** Traditional Dynamic Application Security Testing (DAST) and Static Application Security Testing (SAST) tools rely heavily on static signatures. They lack contextual awareness, generate a high volume of false positives, and cannot chain multiple low-severity vulnerabilities to demonstrate actual risk.
* **Scalability of Manual Pentesting:** Manual penetration testing is highly effective but time-consuming, expensive, and difficult to scale across continuous deployment pipelines. Security teams are overwhelmed by alert fatigue and lack the bandwidth to perform deep, contextual testing on every release.

## 2. Impact of the Problem and Financial Damage
The inability to rapidly identify and contextually verify vulnerabilities leaves enterprise attack surfaces exposed for extended periods. 
* **Financial Devastation:** According to industry reports, the global average cost of a data breach exceeds $4.45 million. Unpatched vulnerabilities and zero-day exploits cost the global economy billions annually in intellectual property theft, regulatory fines, and operational downtime.
* **Resource Drain:** Security Operations Center (SOC) analysts and offensive security teams waste thousands of billable hours annually manually verifying false positives generated by legacy scanners, delaying critical remediation efforts.

## 3. Proposed Solution
AutoPent introduces "Agentic AI" into the offensive security lifecycle. Instead of merely parsing outputs, the framework uses an AI brain to dynamically reason, adapt, and execute attacks.

* **Context-Aware Reconnaissance:** Bypasses standard signature detection by understanding the logical flow of web applications and APIs.
* **Intelligent Payload Generation:** Dynamically crafts context-specific SQL Injection (SQLi), Cross-Site Scripting (XSS), and Local File Inclusion (LFI) payloads based on real-time HTTP responses, rather than relying on a static wordlist.
* **Autonomous Exploit Chaining:** The AI evaluates isolated findings and maps logical exploit paths, chaining minor misconfigurations to achieve high-impact compromises like Remote Code Execution (RCE).
* **Active Verification:** Integrates directly with the Metasploit RPC interface to safely trigger exploits and open reverse shells, providing indisputable proof of vulnerability and eliminating false positives.
* **DevSecOps Remediation:** Closes the feedback loop by generating language-specific, developer-ready code patches for every verified vulnerability.

## 4. Architecture Overview
AutoPent operates on a modular, multi-threaded architecture separated into distinct logical engines:
* **Recon Engine:** Performs "Zero-API" OSINT, including WHOIS lookups, DNS record harvesting, and `crt.sh` subdomain enumeration.
* **Orchestration Layer:** Acts as the central nervous system, routing data between the AI brain, the network tools, and the user interface.
* **Deep Fuzzer Engine:** Crawls the application state, extracting hidden parameters and forms for targeted injection.
* **Exploitation Bridge:** A secure (SSL/TLS) interface connecting the Python logic to the Metasploit (`msfrpcd`) backend.
* **Reporting Engine:** Compiles telemetry, attack graphs, and remediation code into professional PDF documentation.

## 5. Project Structure and Key Features
* `recon.py`: Handles stealthy, external intelligence gathering without triggering rate limits.
* `ai_engine.py`: Integrates with Google Gemini 2.5 Flash to analyze data and formulate attack strategies.
* `intelligent_fuzzer.py`: Executes deep crawling and payload injection logic.
* `exploitation.py`: Manages the Metasploit RPC client connections, handling SSL handshakes and session verification.
* `autopent.py` / `app.py`: The main execution scripts featuring a PyQt5 Command and Control (C2) graphical interface.
* `reporting.py`: Generates the final DevSecOps PDF reports using ReportLab.

## 7. Real-World Examples Used
During development and testing, AutoPent was deployed against standard, legally permissible vulnerable environments (such as ITSEC Games and local DVWA instances). 
* **Scenario:** The Recon Engine identified a hidden parameter (`userinfo.php?user=admin`). 
* **Execution:** The Intelligent Fuzzer recognized it as a database reflection point, and the AI generated an obfuscated SQLi payload to bypass basic filters. 
* **Verification:** The Exploitation Bridge successfully commanded Metasploit to verify the injection point, transitioning a theoretical vulnerability into a confirmed exploit path.

## 8. Outcomes of the Project
* **Drastic Time Reduction:** Condensed the reconnaissance and initial vulnerability mapping phases from several hours to a matter of minutes.
* **Zero False Positives:** By actively integrating with Metasploit to attempt session creation, the framework ensures that only genuinely exploitable vulnerabilities are reported as critical.
* **Developer Enablement:** Reduced the friction between security teams and software engineers by providing exact, copy-pasteable secure code replacements alongside the vulnerability report.

## 9. How to Use the Project

**Disclaimer:** AutoPent is strictly for educational research and authorized penetration testing. Ensure you have explicit permission before scanning any target.

### 1. Clone the Repository
`git clone https://github.com/abhinavmehra2004/hackbyte4.0_AutoPenterS.git`
`cd hackbyte4.0_AutoPenterS`

### 2. Install Dependencies
`pip install -r requirements.txt`

### 3. Environment Setup
Create a `.env` file in the root directory. **Do not commit this file to version control.**

GEMINI_API_KEY=your_api_key<br>

METASPLOIT_RPC_HOST=0.0.0.0

METASPLOIT_RPC_PORT=55553

METASPLOIT_RPC_PASS=your_secure_password

LHOST=your_local_ip

### 4. Start Metasploit RPC
Open a separate terminal and run:
`msfrpcd -P your_secure_password -n -f -a 0.0.0.0 -p 55553`

### 5. Launch the Application
`python3 autopent.py`
   ## 10. Workflow Overview
1. **Target Initialization:** The user inputs a target URL or IP into the C2 interface.
2. **Phase 1: OSINT & Recon:** The system performs DNS harvesting, port mapping, and subdomain enumeration.
3. **Phase 2: Strategy Formulation:** Telemetry is sent to the AI engine to build a customized threat model for the target.
4. **Phase 3: Deep Fuzzing:** The crawler extracts parameters and injects context-aware payloads.
5. **Phase 4: Exploitation Verification:** The AI identifies vulnerable pathways and triggers Metasploit to execute specific modules.
6. **Phase 5: Remediation Reporting:** All data, successful shell session IDs, and AI-generated code patches are compiled into a final PDF.

## 11. Use Cases
* **Red Team Augmentation:** Automating the initial access and enumeration phases of a Red Team engagement, allowing human operators to focus on lateral movement and complex logic flaws.
* **DevSecOps Integration:** Running automated, context-aware security assessments within CI/CD pipelines before code reaches production environments.
* **SME Security Audits:** Providing enterprise-grade vulnerability assessments for small to medium-sized businesses that lack the budget for full-time offensive security personnel.
